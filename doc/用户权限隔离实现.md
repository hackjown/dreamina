# 用户权限隔离功能实现总结

## 问题描述

原有系统中，`projects` 和 `tasks` 表没有 `user_id` 字段，导致：
- 所有用户可以看到所有项目和任务
- 新注册用户可以看到管理员创建的项目和任务
- 没有用户数据隔离机制

## 解决方案

为 `projects` 和 `tasks` 表添加 `user_id` 字段，实现用户数据隔离：
- 普通用户只能查看和管理自己的项目和任务
- 管理员可以查看所有数据

## 修改内容

### 1. 数据库迁移

**文件**: `server/database/migrations/20260326_add_user_id_to_projects_and_tasks.sql`

```sql
-- 为 projects 表添加 user_id 字段
ALTER TABLE projects ADD COLUMN user_id INTEGER;

-- 为 tasks 表添加 user_id 字段
ALTER TABLE tasks ADD COLUMN user_id INTEGER;

-- 将现有数据标记为 admin 用户 (id=1)
UPDATE projects SET user_id = 1 WHERE user_id IS NULL;
UPDATE tasks SET user_id = 1 WHERE user_id IS NULL;

-- 创建索引
CREATE INDEX IF NOT EXISTS idx_projects_user_id ON projects(user_id);
CREATE INDEX IF NOT EXISTS idx_tasks_user_id ON tasks(user_id);
CREATE INDEX IF NOT EXISTS idx_tasks_user_id_project_id ON tasks(user_id, project_id);
```

### 2. Schema 更新

**文件**: `server/database/schema.sql`

- `projects` 表添加 `user_id INTEGER NOT NULL` 字段
- `tasks` 表添加 `user_id INTEGER NOT NULL` 字段

## 权限规则

### 普通用户 (role = 'user')
- 只能查看自己的项目
- 只能在自己的项目中创建任务
- 只能查看自己的任务和下载记录
- 只能修改/删除自己的项目

### 管理员 (role = 'admin')
- 可以查看所有项目
- 可以在任何项目中创建任务
- 可以查看所有任务和下载记录
- 可以修改/删除任何项目

## 数据流向

```
用户请求 → authenticate 中间件 → req.user.id, req.user.role
         ↓
API 端点 → 判断 isAdmin → 调用服务层
         ↓
服务层 → 根据 isAdmin 和 userId 过滤数据
         ↓
数据库查询 → WHERE user_id = ? (非管理员)
```

3. **数据创建测试**
   - 普通用户创建项目

## 注意事项

1. **现有数据归属**: 迁移脚本将现有数据全部归属于 admin 用户 (id=1)
2. **默认项目创建**: `/api/generate-video` 会自动创建"默认项目"，归属于当前用户
3. **向后兼容**: 服务层函数添加了可选参数，保持向后兼容
