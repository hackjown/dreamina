## 2. 文档目标

本 PRD 以当前本地代码实现为准，描述已经落地或明确受当前架构约束的产品能力。它不再沿用旧 GitHub 时代“单页面 + 单 SessionID + 弱权限隔离”的产品叙述。

## 3. 产品定位

Seedance 2.0 是一个面向内容创作者、设计师、营销人员的 AI 视频生成应用。用户可以通过上传参考图、填写提示词、选择模型与参数，在网页端提交视频生成任务，并管理项目、批量任务和下载结果。

当前产品已经从“单次生成工具”演进为“带用户体系、项目管理、批量调度和下载管理的工作台”。

## 4. 当前版本范围

### 4.1 已落地能力

- 邮箱验证码注册、登录、登出、改密
- 积分体系（注册赠送、签到、生成扣减、管理员充值）
- 单任务生成
- 项目管理
- 批量生成管理
- 下载管理
- 管理员后台
- 多即梦 Session 账号管理

### 4.2 当前版本不再使用的旧叙事

以下内容不再代表当前产品：

- 仅依赖 `.env` 或请求体传单个 SessionID
- 普通用户可见管理员项目
- 任务只有单层结构
- 浏览器直接以路径方式下载服务器本地文件
- 登录切换后保留上一个账号的项目 / 设置状态

## 5. 用户角色与权限

### 5.1 普通用户

可执行：

- 注册、登录、改密、登出
- 管理自己的项目、任务、批量和下载记录
- 配置自己的即梦 Session 账号
- 发起单任务和批量生成
- 下载和同步自己的视频结果

不可执行：

- 查看管理员或其他普通用户的项目、任务、批量和下载记录
- 访问管理员后台

### 5.2 管理员

在普通用户能力基础上，还可：

- 查看系统统计
- 管理用户状态
- 修改用户积分
- 重置用户密码
- 查看全量项目 / 任务 / 批量 / 下载记录

## 6. 核心业务规则

### 6.1 平台登录会话

- 登录成功后前端保存平台 Session
- localStorage 键为 `seedance_session_id`
- 请求头使用 `X-Session-ID`
- 前端同时缓存 `seedance_user_cache`

### 6.2 即梦 Session 账号

- 即梦 Session 与平台登录 Session 是两套不同概念
- 即梦 Session 由用户在设置页维护，可保存多个账号
- 第一个账号自动设为默认
- 删除默认账号后自动补一个新的默认账号
- 当前有效 Session 解析顺序为：
  1. `user_default`
  2. `legacy_global`
  3. `env_default`
  4. `none`

### 6.3 数据归属隔离

当前产品必须满足“谁创建，谁可见”的基本约束：

- 普通用户只能看到自己的项目、任务、批量和下载记录
- 管理员拥有全量可见能力
- 批量详情、批量状态、下载列表等衍生能力也必须延续同一归属规则

这是为了避免误删、误操作和跨用户数据泄露。

### 6.4 登录切换后的状态重置

- 当用户退出或登录态失效时，前端全局状态必须清空
- 当切换到另一个账号时，项目、任务、设置必须以新账号身份重新加载
- 不允许保留上一个账号的前端上下文

### 6.5 任务模型

当前任务模型为双层结构：

- `draft`：编辑态任务，承载 prompt、素材、行号、计划输出数
- `output`：从 draft 展开的实际生成任务

批量生成的基础单位是满足条件的 draft 任务，而不是旧版单层任务。

### 6.6 下载链路

下载管理包含两个阶段：

1. 服务端把视频下载到本地目录
2. 浏览器通过一次性 token 获取服务器本地文件

因此当前产品目标不是“前端直接读取本地路径”，而是“安全地把服务器已保存文件转交给浏览器下载”。

## 7. 功能需求

### 7.1 用户体系

| 功能 | 优先级 | 当前要求 |

| 获取当前用户 | P0 | 用于页面初始化与登录态恢复 |

| 修改密码 | P1 | 修改后需重新登录 |

| 功能 | 优先级 | 当前要求 |

| 模型选择 | P0 | 支持 `seedance-2.0` / `seedance-2.0-fast` |
| 参数配置 | P0 | 支持比例、时长、参考模式 |
| 视频结果展示 | P0 | 生成完成后可播放 / 下载 |

### 7.3 项目与任务管理

| 功能 | 优先级 | 当前要求 |

### 7.4 批量生成管理

| 功能 | 优先级 | 当前要求 |

## 8. 关键用户旅程

### 8.1 新用户开始使用

1. 注册账号并完成邮箱验证码验证
2. 自动登录进入系统
3. 在设置页添加一个即梦 Session 账号
4. 创建或使用默认项目
5. 上传素材并发起生成

### 8.2 已登录用户进行批量生成

1. 进入批量管理页选择项目
2. 维护 draft 任务与素材
3. 发起批量生成
4. 查看批量状态、错误与进度
5. 生成完成后在下载管理中查看结果

### 8.3 用户切换账号

1. 当前账号登出或登录态失效
2. 前端清空项目 / 任务 / 设置缓存状态
3. 新账号登录后重新加载自己的数据
4. 不展示上一个账号的任何项目上下文

## 9. 非功能要求

### 9.1 安全

- 平台登录 Session 不出现在 URL 中
- 鉴权头使用 `X-Session-ID`
- 普通用户不可访问他人数据
- 浏览器下载本地文件必须通过 token 化链路
- 管理员操作必须显式受权限约束

### 9.2 可维护性

- 代码和文档应以当前本地实现为准
- 不再沿用旧版 `src/types.ts`、旧 SessionID 优先级、旧任务模型描述
- 文档更新时优先核对 `schema.sql`、`src/types/index.ts`、`App.tsx`、`AppContext.tsx`

## 10. 当前产品约束

- 生成能力依赖即梦平台可用性
- 即梦 Session 失效会直接影响生成、同步与下载刷新
- 批量任务执行受并发参数和即梦侧能力影响
- 单任务与批量任务共用当前用户的即梦 Session 解析结果

## 11. 当前版本结论

当前 Seedance 2.0 已经不是早期“只负责提交一次图生视频请求”的工具，而是一个带有：

- 用户体系
- ownership 隔离
- draft/output 任务模型
- 多 Session 账号管理
- 批量调度
- token 化下载链路

的完整工作台产品。

后续如果继续写文档或做功能规划，必须以这套当前实现为边界，而不是回退到旧 GitHub 文档的叙述方式。
