# Seedance 2.0 数据字典

## 1. 文档信息

| 项目 | 内容 |
|------|------|
| 产品名称 | Seedance 2.0 AI 视频生成 |
| 文档版本 | v1.0.0 |
| 创建日期 | 2026-02-13 |
| 更新日期 | 2026-03-26 |
| 文档状态 | 与当前本地实现对齐 |

## 2. 使用说明

本文档以以下文件为真值来源：

- `server/database/schema.sql`
- `src/types/index.ts`
- `src/services/authService.ts`
- `src/services/settingsService.ts`
- `src/services/downloadService.ts`

当前数据字典不再沿用旧文档中的以下描述：

- `session_id` 是前端 localStorage 登录键
- `seedance_session_id` 是即梦 SessionID
- `email_verification_codes` 只有 `code` 和 `used`
- `tasks` 是旧版单层任务结构
- 系统中不存在 `jimeng_session_accounts` 或 `system_config`

## 3. 数据库表结构

### 3.1 用户与认证模块

#### 3.1.1 `users`

| 字段名 | 类型 | 约束 / 默认值 | 说明 |
|--------|------|---------------|------|
| `id` | INTEGER | PRIMARY KEY AUTOINCREMENT | 用户主键 |
| `email` | TEXT | UNIQUE NOT NULL | 用户邮箱 |
| `password_hash` | TEXT | NOT NULL | 密码哈希 |
| `role` | TEXT | DEFAULT `'user'`，CHECK (`user`/`admin`) | 用户角色 |
| `status` | TEXT | DEFAULT `'active'`，CHECK (`active`/`disabled`) | 账号状态 |
| `credits` | INTEGER | DEFAULT `10` | 积分余额 |
| `daily_check_in` | INTEGER | DEFAULT `0` | 是否已签到的兼容字段 |
| `last_check_in_at` | DATETIME | 可空 | 最后签到时间 |
| `created_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| `updated_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 更新时间 |

补充说明：

- `schema.sql` 会初始化默认管理员账户 `admin@seedance.com`
- 普通用户和管理员共用同一张用户表，通过 `role` 区分能力范围

#### 3.1.2 `sessions`

| 字段名 | 类型 | 约束 / 默认值 | 说明 |
|--------|------|---------------|------|
| `id` | INTEGER | PRIMARY KEY AUTOINCREMENT | 会话主键 |
| `session_id` | TEXT | UNIQUE NOT NULL | 平台登录 Session |
| `user_id` | INTEGER | NOT NULL | 关联 `users.id` |
| `expires_at` | DATETIME | NOT NULL | 过期时间 |
| `created_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

#### 3.1.3 `check_ins`

| 字段名 | 类型 | 约束 / 默认值 | 说明 |
|--------|------|---------------|------|
| `id` | INTEGER | PRIMARY KEY AUTOINCREMENT | 签到记录主键 |
| `user_id` | INTEGER | NOT NULL | 关联用户 |
| `credits_earned` | INTEGER | DEFAULT `2` | 本次签到奖励积分 |
| `created_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 签到时间 |

#### 3.1.4 `email_verification_codes`

| 字段名 | 类型 | 约束 / 默认值 | 说明 |
|--------|------|---------------|------|
| `id` | INTEGER | PRIMARY KEY AUTOINCREMENT | 验证码记录主键 |
| `email` | TEXT | NOT NULL | 目标邮箱 |
| `purpose` | TEXT | DEFAULT `'register'` | 用途：`register` / `login` / `reset_password` / `bind_email` |

| `request_ip` | TEXT | 可空 | 请求来源 IP |
| `consumed_at` | DATETIME | 可空 | 已消费时间 |
| `expires_at` | DATETIME | NOT NULL | 过期时间 |
| `created_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

补充说明：

- 旧文档中的 `code` / `used` 字段描述已失效
- 当前实现已经是增强版验证码表，支持防刷和多业务用途

#### 3.1.5 `system_config`

| 字段名 | 类型 | 约束 / 默认值 | 说明 |
|--------|------|---------------|------|
| `key` | TEXT | PRIMARY KEY | 配置键 |
| `value` | TEXT | 可空 | 配置值 |
| `description` | TEXT | 可空 | 配置说明 |
| `updated_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 更新时间 |

用途：

- 存储 SMTP 等系统级配置
- 不属于用户个人设置

### 3.3 批量、下载与设置模块

#### 3.3.1 `jimeng_session_accounts`

| 字段名 | 类型 | 约束 / 默认值 | 说明 |
|--------|------|---------------|------|
| `id` | INTEGER | PRIMARY KEY AUTOINCREMENT | 账号主键 |
| `user_id` | INTEGER | NOT NULL | 所属用户 |
| `name` | TEXT | DEFAULT `''` | 账号名称 |
| `session_id` | TEXT | NOT NULL | 即梦 SessionID |
| `is_default` | INTEGER | DEFAULT `0` | 是否默认账号 |
| `created_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| `updated_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 更新时间 |

约束：

- `UNIQUE(user_id, session_id)`：同一用户内 SessionID 不可重复
- 唯一默认索引：同一用户最多一个默认账号

业务规则：

- 新用户新增的第一个账号自动成为默认账号
- 删除默认账号后会自动补一个新的默认账号

#### 3.3.2 `settings`

| 字段名 | 类型 | 约束 / 默认值 | 说明 |

| `value` | TEXT | 可空 | 设置值 |
| `updated_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 更新时间 |

当前初始化键包括：

- `model`
- `ratio`
- `duration`
- `reference_mode`
- `download_path`
- `max_concurrent`
- `min_interval`
- `max_interval`
- `session_id`

补充说明：

- `session_id` 仅为 legacy 全局 Session 兼容项
- 当前主流程不再依赖单一全局 SessionID

## 8. 结论

当前 Seedance 2.0 的数据模型核心特征是：

- 用户与平台 Session 分离
- 平台 Session 与即梦 Session 账号分离
- ownership 通过 `projects.user_id` / `tasks.user_id` / `jimeng_session_accounts.user_id` 贯穿
- 任务结构已扩展为 `draft/output`
- 下载链路通过数据库状态 + 本地文件 + token 化浏览器下载组成

后续如需继续维护文档，务必优先核对 `schema.sql` 与 `src/types/index.ts`，不要回退到旧 GitHub 时代的字段解释。
