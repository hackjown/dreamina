## 2. 产品概述
即梦视频管理器是一个面向运营团队的桌面端任务管理工具，用于批量管理即梦视频生成任务。产品通过 Electron 提供本地桌面界面，通过 Playwright 与即梦网页侧能力联动，并支持项目级任务管理、单条/批量生成、定时生成、任务采集和视频下载。

从现有实现看，产品重点不是通用视频编辑，而是提升运营团队在“批量提示词投放、过程跟踪、结果采集、本地归档”场景下的执行效率。

## 3. 业务目标
### 3.1 核心目标
1. 降低运营团队批量提交即梦视频任务的人工重复操作。
2. 提供项目化的任务组织方式，方便按活动、专题、账号或批次管理视频生产任务。
3. 支持任务状态跟踪、补采集和本地下载，减少因网页端超时或页面关闭造成的结果丢失。
4. 通过统一的模型、比例、时长、并发等设置，提升运营执行标准化程度。

### 3.2 成功标准
1. 运营人员可在单个项目下批量维护多条视频生成任务。
2. 支持单任务生成与批量生成，并可配置随机间隔和重复次数。
3. 生成结果可被回收、预览、下载，并按项目维度规范化保存到本地。
4. Session ID 可测试，基础配置可持久化保存。

## 4. 用户与使用场景
### 4.1 目标用户
- 运营团队成员
- 内容生产人员
- 需要批量管理视频生成任务的执行人员

### 4.2 典型使用场景
1. **活动素材批量生产**
   - 运营按活动建立项目。
   - 为每条创意建立任务并填写提示词、参考图、音频。
   - 批量提交生成并等待产出。

2. **定时批次投放**
   - 运营预先配置任务范围、重复次数与启动时间。
   - 在指定时间触发定时生成，适配夜间或固定时段执行场景。

3. **失败任务补采集**
   - 当网页端生成已完成但桌面端未拿到结果时，运营可手动批量采集历史任务结果。

4. **结果归档与交付**
   - 运营预览生成结果，打开所在目录或下载视频到本地项目文件夹，用于后续剪辑、审片或分发。

### 5.2 明显未闭环或未完全兑现的需求
以下内容在需求草稿中出现，但从代码实现看尚未完全闭环。

1. **真正的并发批量执行**
   - 需求草稿提到“设置并发数和时间，最大并发数任务结束，自动继续并发”。
   - 当前实现虽然保存了 `concurrent` 配置，但 `processBatch` 仍按单序列顺序执行，并未形成多任务并行调度。
   - 依据：`src/main/index.ts:446`-`src/main/index.ts:464`

2. **音频素材实际参与生成**
   - 前端已支持添加音频文件，但主流程未见将音频注入生成请求的逻辑。
   - 依据：`src/renderer/index.html:682`、`src/main/index.ts:213`-`src/main/index.ts:368`

3. **自动下载到本地**
   - 需求草稿写明“生成视频自动保存到本地”，但当前代码表现为生成完成后可手动下载，未见自动下载闭环。
   - 依据：`src/renderer/index.html:414`-`src/renderer/index.html:435`

## 6. 关键业务流程
### 6.1 项目创建与任务准备
1. 用户创建项目并指定任务数量。
2. 系统初始化多个空任务。
3. 用户逐条填写提示词，并可附加图片/音频素材。
4. 用户保存项目。

### 6.2 单任务生成流程
1. 用户填写提示词并确认 Session ID 已配置。
2. 前端调用 `generate-video` IPC。
3. 主进程初始化浏览器上下文并注入 Cookie。
4. 如有参考图，系统先上传素材并获得 URI。
5. 系统提交生成请求并轮询结果。
6. 返回视频 URL 和 historyId，界面展示完成状态。

### 6.3 批量生成流程
1. 用户从项目右键菜单打开批量生成或定时生成。
2. 选择任务范围与重复次数。
3. 系统启动批处理，按顺序执行每个任务。
4. 每轮和每项任务之间有随机等待。
5. 前端接收状态更新并刷新显示。

### 6.4 结果采集与归档流程
1. 用户对未完成但已存在 historyId 的任务执行采集。
2. 系统根据 historyId 查询结果。
3. 成功后将视频 URL 挂到任务结果列表。
4. 用户可预览视频或手动下载到本地项目目录。

## 7. 非功能需求
### 7.1 可用性
- 提供桌面端图形界面，降低技术门槛。
- 支持深色/浅色主题切换。
- 项目与设置具备本地持久化能力。

### 7.2 性能
- 现阶段更关注稳定执行而非极致吞吐。
- 批处理采用轮询和延时机制，降低连续请求频率。

### 7.3 可靠性
- 对浏览器初始化失败、上传失败、生成失败、采集失败提供错误回传。
- 支持中途取消任务。
- 支持失败后人工补采集。

### 7.4 安全性
- Session ID 保存在本地设置中，用于注入即梦站点 Cookie。
- 当前未见加密存储、脱敏展示或权限隔离机制。

## 9. 已知问题与改进方向
1. 并发参数尚未真正生效，可能导致用户预期与实际不一致。
2. 生成、采集、下载均依赖第三方网页接口和页面行为，稳定性受外部平台影响。
3. 前端启用了 `nodeIntegration: true` 且 `contextIsolation: false`，存在安全风险，但这更属于架构设计层面的技术债。
4. 数据结构未显式定义 TypeScript 类型，后续维护成本较高。
5. 目前缺少测试代码与自动化校验机制。

## 10. MVP 验收标准
1. 可创建、保存、删除项目。
2. 每个项目可维护多条任务，任务支持提示词、参考图、音频信息录入。
3. 可通过 Session ID 建立与即梦站点的工作连接。
4. 支持单条任务生成、批量生成、定时生成。
5. 任务状态在 UI 中可追踪。
6. 已完成或可回收的任务可采集视频结果。
7. 视频可预览、下载并在本地目录中查看。
